# Viewing the cluster list
{: #view-cluster-list}

Open the **Clusters** tab in your Fusion as a Service deployment to see all associated clusters.
{: shortdesc}

## Before you begin
{: #view-cluster-list-prereqs}

Before you view the cluster list, make sure that you can open the deployment in the {{site.data.keyword.cloud_notm}} console.

## Open the cluster list
{: #access-cluster-list}

1. Open your Fusion as a Service deployment.
2. In the left navigation, click **Clusters**.

The **Clusters** tab opens and displays a table showing all clusters that are associated with the deployment.

## Information shown in the cluster list
{: #cluster-list-info}

The cluster list displays a table with the following columns for each cluster:

Name
:   The cluster name. Click the name to open cluster details.

Status
:   The current operational status of the cluster.

vCPU
:   The number of virtual CPUs allocated to the cluster.

Memory
:   The amount of memory allocated to the cluster.

Storage
:   The storage quota and current usage, if Fusion services are enabled for the cluster.

Additional indicators such as warnings or health status may also appear in the list to help you identify clusters that need attention.

## Why to use the cluster list
{: #filter-sort-clusters}

Use the cluster list to do the following tasks:

- Identify which clusters are associated with the deployment.
- Review the current status of each cluster.
- Find clusters that need attention.
- Open a cluster to review more detailed information.
